# 需求完成度核查（非集成范围）

日期：2026-04-21

来源：`docs/需求.md`

## 1. 核查口径

本次按你的要求，只核查“非集成范围”。以下内容不作为本次是否完成的阻塞项：

- CRM、组织架构系统、OA/审批流、财务系统、电子签章等外部系统对接
- 外部招标平台、第三方商机服务、公开中标结果抓取等外部数据源联调
- 邮件、单点登录、外部开放 API 等集成能力

但外部集成剔除后，系统自身仍需要满足三层闭环：

- `前端`：有页面、菜单或明确交互入口，且不是纯占位
- `前端 API`：`src/api` 中存在真实 HTTP 接口封装
- `后端 Controller`：`backend/src/main/java` 中存在对应 REST Controller

判定规则：

- `已完成`：前端 + 前端 API + 后端 Controller 都具备，主链路能对上
- `部分完成`：三层中有缺口，或字段/维度无法支撑需求验收
- `不纳入本次`：纯集成项，按本次要求剔除

## 2. 明确结论

**结论：非集成范围尚未全部完成。**

《需求.md》共梳理 `29` 个功能点口径，其中：

- `4` 项为纯集成项，本次不纳入
- `25` 项进入非集成核查
- `22` 项已完成
- `3` 项部分完成

本次核查时已顺手补齐一个实际断点：

- `投标结果闭环` 页面已经有交互、后端也有 controller，但 `bidResultsApi` 缺少登记、更新、确认补录、附件绑定、提醒上传回写、竞争对手中标记录等命令接口。
- 已补齐：`src/api/modules/bidResults.js`
- 已补测试：`src/api/modules/bidResults.spec.js`

剩余不能宣称“全部完成”的项目是：

1. `一站式流程发起`：合同借阅申请未形成独立闭环。
2. `外部标讯获取与入库` 的非集成部分：标讯库基础 CRUD 有，但后端标讯模型缺地区、行业、采购方等字段，无法完整支撑需求里的多维检索。
3. `多维深度分析报表`：区域、产品线、销售团队、竞争对手维度已做，客户类型维度未见前端/API/controller 闭环。

## 4. 未完成项的明确缺口

### 4.1 一站式流程发起

已完成：

- 标书支持申请：工作台弹窗提交审批。
- 资质借阅：资质库借阅/归还链路。
- 投标费用申请：费用管理申请/审批/支付链路。

缺口：

- 未见独立的合同借阅申请页面/弹窗。
- 未见合同借阅专用前端 API。
- 未见合同借阅专用后端 controller。

要达到完成：

- 增加合同借阅申请入口，或明确合同文件归属到现有资质/模板借阅口径。
- 若是独立流程，需要补 `前端表单 + contractBorrowApi + ContractBorrowController`。

### 4.2 标讯获取与入库（非集成部分）

已完成：

- 标讯中心页面。
- 人工录入弹窗。
- `tendersApi` 基础 CRUD。
- `TenderController` 基础 CRUD。
- CEB 抓取入口存在，但外部抓取联调不纳入本次。

缺口：

- 后端 `Tender` 实体和 `TenderDTO/TenderRequest` 目前主要包含 `title/source/budget/deadline/status/aiScore/riskLevel`。
- 需求要求标讯库支持按时间、地区、采购方、关键词等检索；其中地区、采购方、行业等字段未在后端标讯模型中形成真实持久化字段。
- 前端页面虽然展示/录入了地区、采购方、行业等信息，但这些字段无法完整落到当前后端 controller 的真实合同中。

要达到完成：

- 后端补齐 `region/industry/purchaser/publishDate` 等字段和迁移。
- `TenderController` 支持查询参数过滤。
- `tendersApi.getList()` 改为真实查询参数透传。
- 前端检索项与后端字段保持一致。

### 4.3 多维深度分析报表

已完成：

- 区域分析。
- 产品线分析。
- 销售团队/团队绩效下钻。
- 竞争对手分析。
- ROI 与评分分析能力在项目 AI 能力中已有基础。

缺口：

- 未见客户类型维度的图表、查询 API 或后端 controller。
- 当前 `DashboardController` 暴露了 overview、trends、competitors、regions、product-lines、drilldown 等，但没有 customer-types 类接口。

要达到完成：

- 定义客户类型字段来源。
- 后端增加客户类型统计接口，例如 `/api/analytics/customer-types`。
- 前端 `dashboardApi` 增加对应封装。
- 数据分析页增加客户类型维度图表或筛选/下钻入口。
