# 交付阻塞清单（研发排期版）

更新时间：2026-04-15

## 使用说明

- 本清单只记录高置信度阻塞项：明确未接真实后端、按钮是假动作、页面仍依赖 mock、或主测试链路未打通。
- `风险等级` 说明：
  - `P0`：会直接影响客户验收或造成“点了没反应 / 功能不真实”。
  - `P1`：核心链路可继续走，但会暴露能力不完整、结果不可信或明显半成品状态。
  - `P2`：可延后，但建议在正式扩量前补齐。
- `是否必须交付前解决` 说明：
  - `是`：建议作为本次交付门禁。
  - `否`：可进入后续迭代，但前提是入口不暴露给客户，或有明确口径。

## 建议排期顺序

### 第一批：必须在交付前清零

- Header：搜索、通知、个人中心
- 系统设置：用户与角色相关空按钮
- 费用台账：导出、详情、费用申请
- 账户管理：详情、新增账户
- 中标结果：同步、同步、提醒、确认、忽略、详情
- 项目详情：演示任务、演示文档、演示归档、演示提醒
- 文档编辑器：演示预览、演示文档包、演示归档
- 客户机会中心：整页是否交付必须立即定口径
- 标准 E2E 主链路恢复

### 第二批：交付前至少保证不暴露占位态

- Dashboard：产品线分析、下钻明细、导出链路
- 项目创建：评分预览
- 项目详情：AI 检查真实结果
- 工作台：快捷发起入口
- 知识库：模板库、案例库
- Bidding AIAnalysis
- BAR 借用弹窗与相关资源页

### 第三批：可放到后续迭代，但需先隐藏入口

- 模板库版本历史
- 其他仅影响深度使用、不影响本次客户首轮验收的扩展功能

## 排期建议

- 建议每一行都补一个责任人、目标完成时间、联调状态。
- 交付前最终 gate 建议只看两件事：
  - 所有 `是否必须交付前解决 = 是` 的项全部关闭。
  - API 模式下主链路 E2E 跑通，且不再出现演示/暂未接入提示。
