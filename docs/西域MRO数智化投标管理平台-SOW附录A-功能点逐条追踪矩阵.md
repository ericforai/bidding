# 西域MRO数智化投标管理平台
## SOW 附录 A：功能点逐条追踪矩阵（29 项口径版）

## 1. 使用说明

本矩阵用于将客户需求、功能响应表、SOW 详细版、实施交付和验收口径进行一一对应，建议作为正式 SOW 的附录 A 使用。

建议用途：
- 作为蓝图阶段需求锁定依据
- 作为开发与测试追踪依据
- 作为 UAT 与最终验收依据
- 作为范围争议时的判定依据

## 2. 与客户需求匹配结论

依据《西域MRO数智化投标管理平台功能响应一览 V1.1》及最新补充要求，本附录已纳入以下关键口径：

- 报表范围按 `6` 张控制
- 业务侧标讯平台地址为 [cebpubservice](http://www.cebpubservice.com/)，按第三方平台 API 方式对接
- 预测信息集成于“市场洞察”中展示
- 项目卡片增加“项目负责人”和“销售负责人”
- 项目负责人支持手工修改，销售负责人同步 CRM 数据
- 组织架构同步来源明确为企业微信
- AI 检查项支持维护和调整

## 3. 口径说明

### 3.1 数量口径说明

客户提供的《西域MRO数智化投标管理平台功能响应一览 V1.1》汇总区写明“核心功能点 `29` 个”，但明细逐条展开时实际列出了 `30` 行功能项。

为保持与客户当前汇总口径一致，本附录采用“`29` 项口径版”编制，处理方式如下：
- 第 `1-28` 项按源表明细逐条保留
- 第 `29` 项将 AI 模块中的“合规性检查”与“文本书写质量辅助”合并为一个验收条目“AI 标书检查与文本质量辅助”

建议：
- 正式提交客户版时保留本口径说明
- 在蓝图确认会或需求确认书中，与客户书面确认“29 项”还是“30 项”最终验收口径

### 3.2 字段说明

- `来源需求`：来自客户功能响应表的原始功能点
- `建设内容`：项目内需交付的能力说明
- `交付方式`：标准配置、接口集成、定制开发、报表开发等实现方式
- `主要交付物/落点`：系统页面、配置成果、接口文档或报表对象
- `验收口径`：客户验收时建议使用的判定标准
- `边界/前提`：容易引发争议的限制条件、依赖条件或非默认范围

### 3.3 第三方平台与费用前提

以下事项建议在正式 SOW 正文中同步列为项目通用前提：

- 第三方平台 API、账号、授权、测试环境、网络白名单及联调条件由甲方或甲方协调第三方提供
- 第三方平台接口调用费用、账号采购费用、订阅费用、AI Token 费用不包含在本期实施报价内，除非商务合同另有明确约定
- 如第三方平台接口能力、字段质量、频率限制或授权策略发生变化，双方应基于实际条件调整接口方案、交付计划与验收安排
